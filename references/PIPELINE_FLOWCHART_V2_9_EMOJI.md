## 🎯 v2.9 核心改进（在 v2.8 骨架上吸收，而不是重写替代）

### 1. main 直接编排恢复为默认模式
- 不再让 launcher 脚本 成为默认入口
- 经验内化为规则，不再以脚本形式绑死流程

### 2. Lite 快车道成为“插层”，不是平行体系
- `founder-office-hours` + `autoplan-lite` 只在合适的 L2 使用
- 不会抹掉原有 Full 主链

### 3. review-gate 前移成硬门
- coding announce 不再等价于完成
- 没有 gate 判定，不进入完成态

### 4. 浏览器 QA 正式入链
- UI / workflow / browser-visible 路径有正式 QA 位置
- 不再临时补一个测试就算了

### 5. Release 复盘 成为按需 步骤 8
- 有学习价值才执行
- 交付后沉淀更自然，不强迫每次都跑

---

## 📢 通知规范

### 可靠通知 owner
- 永远是 `main`

### 群 / DM 分工
- **职能群**：步骤级通知
- **监控群**：完整审计链
- **晨星DM**：只用于最终结果 / 需介入失败 / 长任务摘要

### 硬规则
- 不要把 sub-agent 自推当成可靠送达
- 不要把 coding announce 当成完成证明
- 如果任务本身就在晨星DM中进行，不额外重复主动 DM
- `message(action="send")` 单发必须使用 `target`，不要使用 `targets`
- 同时发多个群时，必须按单目标串行发送多次；`targets` 只允许用于 `action="broadcast"`
- 禁止把多个群 id 塞进一次 `send`
